# Cursor 作業指示（非公開）

## このドキュメントの目的

- Cursor（AI コーディング支援）に「何を優先し、どこを触り、どう記録するか」を明確にし、迷いと手戻りを減らします。

## 基本ルール

- **公開 Docs と混ぜない**: エンドユーザー向け Docs は `toxbox-site/src/content/docs/`（Starlight）に置く
  - 本 `docs/` 配下は非公開（開発・運用・Cursor 用）
- **ビルド対象の確認**: GitHub Pages のビルド対象は `toxbox-site/` のみ
  - `docs/` やリポジトリ直下の Markdown はデプロイ成果物に含まれません

## 作業の進め方（推奨）

- **現状確認**: `git status` / `git diff` / 関連ファイルを読んでから編集
- **変更の単位**: 目的が同じものを 1 コミットにまとめる（Docs 整理、機能追加など）
- **コミットメッセージ**: 文字化け回避と履歴の可読性のため **英語**を推奨（例: “Split internal docs into docs/”）

## 触って良い/悪い場所（目安）

- **触って良い（非公開）**: `docs/`、リポジトリ直下の補助文書
- **触って良い（サイト）**: `toxbox-site/` 配下（ページ、公開 Docs、設定）
- **注意**: `.github/workflows/deploy.yml` はデプロイに直結するため、小さく・確実に変更する
