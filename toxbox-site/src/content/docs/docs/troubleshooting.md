TOXBOX を使用中に発生する可能性のある問題と、その解決策をまとめています。

## 起動・セットアップ

### プロジェクトが開けない / 起動しない

**症状**: `TOXBOX.toe` をダブルクリックしても TouchDesigner が起動しない、またはエラーが表示される。

**考えられる原因と対処**:

1. **TouchDesigner がインストールされていない**
   - [TouchDesigner 公式サイト](https://derivative.ca/download) からダウンロード・インストール

2. **TouchDesigner のバージョンが古い**
   - TOXBOX は **TouchDesigner 2025.32050** で制作されています
   - 以前のバージョンでは互換性の問題が発生する可能性があります
   - 最新版へのアップデートを推奨

3. **ファイルパスに問題がある**
   - 日本語や特殊文字を含むパスを避ける
   - 例: `C:\VJ\TOXBOX\` (OK) vs `C:\VJ作業\TOXBOX\` (問題の可能性あり)

4. **ファイルが破損している**
   - 再度ダウンロードして解凍し直す

### 起動時にエラーダイアログが表示される

**症状**: TouchDesigner は起動するが、エラーメッセージが表示される。

**対処**:
- エラーメッセージの内容を確認
- バージョン差による互換性エラーの場合は、TouchDesigner をアップデート
- 外部ファイル参照エラーの場合は、フォルダ構成が正しいか確認

## 映像出力

### 画面が真っ黒 / 何も表示されない

**チェック項目**:

1. **シーンが読み込まれているか**
   - Scene Panel にサムネイルが表示されているか確認
   - 表示されていない場合は tox ファイルをドラッグ＆ドロップで読み込む

2. **シーンが選択されているか**
   - シーンスロットをクリックしてアクティブ化する
   - 選択状態のシーンは枠線でハイライトされる

3. **レイヤーフェーダーが上がっているか**
   - フェーダーが 0 だと表示されない
   - フェーダーを上げて透明度を確認

4. **出力ウィンドウが正しいモニターに表示されているか**
   - マルチモニター環境では、出力先を確認

### 特定の tox だけ表示されない

**原因**: tox 自体に問題がある可能性。

**確認方法**:
1. その tox を直接 TouchDesigner で開く
2. tox 内部の `Out TOP` に正しく映像が出力されているか確認
3. エラーや警告がないか確認（赤い文字のオペレーターなど）

### 映像がカクつく / フレームレートが低い

パフォーマンスセクションを参照してください。

## パフォーマンス

### 動作が重い / カクつく

**即効性のある対処**:

1. **同時表示レイヤー数を減らす**
   - 4 レイヤーすべてを表示すると負荷が 4 倍
   - 使っていないレイヤーのフェーダーを 0 に

2. **ポストエフェクトを減らす**
   - 特に Blur, Bloom, Feedback は負荷が高い
   - 不要なエフェクトをオフに

3. **出力解像度を下げる**
   - 4K → 1080p、1080p → 720p など

**tox 側の最適化**:

1. **重い tox を特定する**
   - レイヤーを 1 つずつ表示して、どの tox が重いか確認
   - 重い tox は最適化するか、使用を控える

2. **tox 内部を確認**
   - 不要に高い解像度で処理していないか
   - Heavy なオペレーター（Blur TOP など）を多用していないか

**システム設定**:

1. **他のアプリケーションを終了**
   - ブラウザ、動画再生ソフトなどを閉じる

2. **GPU ドライバーを更新**
   - 最新の GPU ドライバーをインストール

3. **TouchDesigner の Performance Monitor を確認**
   - `Dialogs` → `Performance Monitor` で負荷の内訳を確認

### フレームレートを確認する方法

TouchDesigner のメニューから `Dialogs` → `Performance Monitor` を開くと、現在のフレームレートと各オペレーターの負荷を確認できます。

## オーディオ

### 音声が入力されない

**チェック項目**:

1. **正しいデバイスが選択されているか**
   - Audio Analysis Panel で `Audio Device In` を確認
   - システムの既定のデバイスと一致しているか確認

2. **デバイスがミュートされていないか**
   - Windows の音量ミキサーでミュート状態を確認
   - マイク入力の場合、プライバシー設定でアクセスが許可されているか確認

3. **ループバック設定**
   - PC の再生音を入力したい場合は、仮想オーディオデバイスが必要
   - Windows: ステレオミキサー、VB-CABLE など
   - macOS: BlackHole、Loopback など

### オーディオ反応が遅れる

**原因**: オーディオバッファサイズが大きい。

**対処**:
1. TouchDesigner のメニューから `Edit` → `Preferences` → `Audio`
2. `Buffer Size` を小さい値に設定（256 や 512）
3. 小さすぎると音が途切れる場合があるので調整

## MIDI

### MIDI コントローラーが認識されない

**チェック項目**:

1. **物理接続を確認**
   - USB ケーブルが正しく接続されているか
   - 別の USB ポートを試す

2. **ドライバーがインストールされているか**
   - コントローラーのメーカーサイトから最新ドライバーをインストール

3. **TouchDesigner での認識を確認**
   - `Dialogs` → `MIDI Device Mapper` でデバイスが表示されているか確認
   - 表示されていない場合は TouchDesigner を再起動

4. **他のアプリケーションとの競合**
   - 他の DAW やソフトが MIDI デバイスを占有していないか確認

### マッピングが効かない

**チェック項目**:

1. **Learn モードが有効か**
   - MIDI Learn モードがオンになっているか確認

2. **正しい MIDI チャンネル**
   - コントローラーが送信している MIDI チャンネルを確認

3. **マッピング済みかどうか**
   - 既存のマッピングが干渉していないか確認

## tox 関連

### tox が読み込めない

**原因と対処**:

1. **TouchDesigner のバージョン差**
   - より新しいバージョンで作成された tox は開けない場合がある
   - tox 作成者に確認、または TouchDesigner をアップデート

2. **ファイルが破損している**
   - 元のファイルを再入手

3. **外部依存がある**
   - tox が外部ファイル（画像、動画など）に依存している場合、そのファイルが見つからないとエラーになる

### パラメータがパネルに表示されない

**原因**: tox がパラメータパネル対応の形式で作成されていない。

**対処**:
- tox のカスタムパラメータを確認
- [tox作成ガイド](../creating-tox/) を参照して形式を確認

## バージョン互換性

### 旧バージョンで作った tox が動かない

TouchDesigner はバージョン間での完全な互換性が保証されていません。

**対処**:
1. tox を作成した TouchDesigner バージョンを確認
2. 同じバージョンで開いて、新しいバージョン向けに保存し直す
3. それでも問題がある場合は、tox 内部のエラーを修正

### TOXBOX のバージョンアップ後に問題が発生

**対処**:
1. リリースノートで変更点を確認
2. 設定やマッピングが初期化されている場合は再設定
3. 問題が解決しない場合は、以前のバージョンの TOXBOX を使用

## その他

### プロジェクトを保存できない

**原因**:
- ファイルが読み取り専用になっている
- ディスク容量が不足している
- ファイルパスが長すぎる

**対処**:
1. ファイルのプロパティで読み取り専用を解除
2. ディスク空き容量を確認
3. より短いパスに移動

### 設定が保存されない

TOXBOX の設定（MIDI マッピングなど）は、プロジェクトファイル（.toe）に保存されます。

- `File` → `Save` でプロジェクトを保存
- 上書き保存で設定が反映される

---

## 問題が解決しない場合

1. **TouchDesigner を再起動**
   - 多くの問題は再起動で解決する

2. **PC を再起動**
   - デバイスの認識問題などはシステム再起動で解決することがある

3. **最新バージョンを使用**
   - TouchDesigner と TOXBOX の両方を最新版に更新

4. **サポートに連絡**
   - BOOTH の購入ページからお問い合わせ
   - 問題の詳細（症状、エラーメッセージ、環境）を添えて連絡
